# Arcanaglyph: Development Roadmap (MVP)

Этот документ — пошаговый план создания MVP приложения Arcanaglyph на Rust.
Каждый этап представляет собой логический блок с задачами, которые можно отмечать по мере выполнения.

---

## Phase 0: Foundation & Environment Setup

**Цель:** Подготовить рабочее окружение и структуру проекта. Это самый важный этап для знакомства с экосистемой.

- [x] **1. Инициализация проекта с Cargo:**
  - Вы уже создали директорию. Убедитесь, что внутри выполнен `cargo init` или `cargo new .`.
  - Это создаст `src/main.rs` и `Cargo.toml`.

- [x] **2. Изучение `Cargo.toml`:**
  - Откройте `Cargo.toml`. Это ваш `pyproject.toml` или `package.json`.
  - Ознакомьтесь со структурой секций `[package]` и `[dependencies]`.

- [x] **3. Настройка IDE / Редактора:**
  - Установите расширение `rust-analyzer` для вашего редактора (VS Code, JetBrains и др.).
  - Это обеспечит автодополнение, проверку ошибок и навигацию по коду, что критически важно для изучения.

- [x] **4. Установка системных зависимостей для Ubuntu:**
  - Для работы с аудио, GUI и другими системными вещами Rust часто требует системные библиотеки.
  - Выполните в терминале:

    ```bash
    sudo apt-get update && sudo apt-get install build-essential libasound2-dev libgtk-3-dev libwebkit2gtk-4.1-dev
    ```

  - Это закроет большинство будущих потребностей для GUI и аудио.

## Phase 1: Core Logic - The "Command-Line" Proof of Concept

**Цель:** Реализовать основную механику "аудио -> текст" в простом консольном приложении,
игнорируя пока GUI и системную интеграцию.

- [x] **1. Захват аудио с микрофона:**
  - **Задача:** Написать код, который захватывает аудио с микрофона по умолчанию и сохраняет его в буфер или файл.
  - **Крейты на выбор:**
    - [`cpal`](https://crates.io/crates/cpal): Низкоуровневый, кросс-платформенный стандарт для работы с аудио.
    - [`rodio`](https://crates.io/crates/rodio): Более высокоуровневая обертка над `cpal`, может быть проще для начала.

- [x] **2. Локальная транскрибация аудио:**
  - **Задача:** Взять сохраненные аудиоданные и передать их в локальную модель для получения текста.
  - **Крейты на выбор:**
    - [`whisper-rs`](https://crates.io/crates/whisper-rs): Rust-биндинги к `whisper.cpp`. Самый прямой путь.
    - [`candle`](https://github.com/huggingface/candle): ML фреймворк от Hugging Face на Rust. Более сложный,
      но и более гибкий путь, если захотите в будущем менять модели.

- [x] **3. Соединение логики:**
  - **Задача:** Объединить два предыдущих шага в одной программе, которая слушает, а потом печатает текст в консоль.
  - На этом этапе у вас будет консольное приложение, доказывающее работоспособность основной идеи.

## Phase 2: Building the Desktop Application Shell

**Цель:** Обернуть нашу консольную логику в минимальное десктопное приложение.

- [x] **1. Выбор GUI фреймворка:**
  - **Задача:** Выбрать и интегрировать фреймворк для создания окна приложения или иконки в трее.
  - **Крейты на выбор:**
    - [`Tauri`](https://tauri.app/): Рекомендуется. Легковесный, позволяет делать UI на веб-технологиях
      (HTML/JS/CSS), а всю логику на Rust. Идеально для быстрого старта.
    - [`egui`](https://crates.io/crates/egui): "Чистый" Rust, immediate mode GUI. Отлично подходит для простых
      инструментов и оверлеев.
    - [`iced`](https://crates.io/crates/iced): "Чистый" Rust, Elm-архитектура. Более структурированный подход.

- [x] **2. Создание базового интерфейса:**
  - **Задача:** Создать окно (или просто иконку в трее) с минимальным набором элементов.
  - Например, кнопка "Начать запись" и текстовое поле для вывода результата транскрибации.

- [x] **3. Интеграция Core Logic:**
  - **Задача:** Перенести логику из Phase 1 в GUI-приложение. Нажатие кнопки должно запускать
    процесс записи и транскрибации.

## Phase 3: System Integration - The "Magic"

**Цель:** Превратить приложение в удобный системный инструмент, который работает в фоне.

- [ ] **1. Реализация глобальных горячих клавиш:**
  - **Задача:** Зарегистрировать комбинацию клавиш, которая будет работать из любого приложения.
  - **Крейты на выбор:**
    - [`global-hotkey`](https://crates.io/crates/global-hotkey): Специализированный крейт для этой задачи.
    - [`rdev`](https://crates.io/crates/rdev): Более мощный крейт для прослушивания всех глобальных событий
      (мышь, клавиатура).

- [ ] **2. Реализация симуляции ввода:**
  - **Задача:** Написать код, который программно вставляет текст в то место, где сейчас находится курсор.
  - **Крейты на выбор:**
    - [`enigo`](https://crates.io/crates/enigo): Простой и популярный крейт для симуляции ввода с мыши и клавиатуры.
    - [`rdev`](https://crates.io/crates/rdev): Также может и отправлять события, а не только слушать их.

- [ ] **3. Финальная сборка workflow:**
  - **Задача:** Соединить все вместе.
    - `Hotkey 1` -> Начать/продолжить запись.
    - `Hotkey 2` (или та же) -> Остановить запись -> Запустить транскрибацию -> Вставить результат.

---

## Phase 4: Polishing & Packaging

**Цель:** Довести MVP до состояния готового к использованию продукта.

- [ ] **1. Иконка в системном трее:**
  - **Задача:** Добавить иконку в трей для управления приложением (настройки, выход).
  - **Крейты на выбор:**
    - [`tray-icon`](https://crates.io/crates/tray-icon): Кросс-платформенный крейт для иконок в трее.
    - Многие GUI фреймворки (например, `Tauri`) имеют встроенную поддержку трея.

- [ ] **2. Конфигурация:**
  - **Задача:** Позволить пользователю настраивать горячие клавиши и выбирать модель транскрибации через конфиг-файл.
  - **Крейты на выбор:**
    - [`config`](https://crates.io/crates/config): Гибкий крейт для работы с конфигурационными файлами.
    - [`serde`](https://crates.io/crates/serde): Обязательный крейт для любой сериализации/десериализации в Rust.

- [ ] **3. Сборка дистрибутива:**
  - **Задача:** Собрать приложение в виде одного исполняемого файла для Linux и Windows.
  - Используйте `cargo build --release`. Если вы выбрали `Tauri`, команда `cargo tauri build`
    создаст для вас готовые инсталляторы (`.deb`, `.AppImage`, `.msi`).
