# Changelog

Все заметные изменения в проекте документируются в этом файле.
Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/),
версионирование следует [Semantic Versioning](https://semver.org/lang/ru/).

## [Unreleased]

## [1.10.1] - 2026-06-16

### Исправлено

- Автозапуск на Windows и macOS теперь применяется и при обновлении: если опция
  была включена раньше, она срабатывает сама при следующем запуске, без повторного
  переключения галочки в настройках.

## [1.10.0] - 2026-06-16

### Добавлено

- Автозапуск при входе в систему теперь работает на Windows и macOS, а не только
  на Linux: на Windows ярлык регистрируется в реестре, на macOS — через LaunchAgent.

## [1.9.5] - 2026-06-16

### Изменено

- Горячие клавиши записи на Linux стали надёжнее: триггер больше не зависит от
  внешней утилиты `nc` и фиксированного сетевого порта — нажатие клавиши общается
  с приложением через локальный сокет напрямую. Уходит и установка вспомогательных
  скриптов; при обновлении горячие клавиши перенастраиваются автоматически.

### Исправлено

- Запуск в свёрнутом виде (настройка «Запускать свёрнуто в трее») снова работает:
  окно больше не появляется при старте, если включена эта опция.
- Автозапуск при входе в систему снова работает после установки: ярлык автозапуска
  обновляется при старте под актуальный путь приложения, а не остаётся со старым.

## [1.9.4] - 2026-06-15

### Исправлено

- Проверка обновлений больше не показывает «Установлена последняя версия», когда
  новая версия на самом деле доступна. Раньше после первой же проверки приложение
  «забывало» уже найденное обновление (ответ сервера «без изменений» сбрасывал его)
  — теперь оно сохраняется, и кнопка «Обновить» появляется как положено.

## [1.9.3] - 2026-06-15

### Исправлено

- Запись по горячей клавише на микрофонах с аппаратным шумоподавлением (например
  Intel Smart Sound) больше не обрывается, если начать молча, и не теряет начало
  фразы. Запись и плашка теперь стартуют сразу по нажатию, а не после первого
  произнесённого слова.

## [1.9.2] - 2026-06-15

### Исправлено

- Обновление больше не «залипает», если закрыть окно терминала установки до её
  завершения: при следующем запуске приложение снова покажет «Доступно
  обновление» и позволит повторить. Раньше баннер «Установка идёт… /
  Перезапустить» оставался навсегда, и обновиться было нельзя даже после
  перезапуска приложения.

## [1.9.1] - 2026-06-14

### Исправлено

- Сборка установщика для macOS (экспериментальная платформа, без подписи)
  теперь компилируется: окно виджета записи использует прозрачность, которая
  на macOS требует приватного API Tauri — он включён. Для Linux и Windows
  поведение не меняется.

## [1.9.0] - 2026-06-14

### Добавлено

- GigaAM v3 RNN-T — новый, более точный движок распознавания русской речи
  (WER ~8.4% против ~9.2% у прежнего CTC-варианта). Выбран по умолчанию и
  стоит первым в списке моделей; прежний CTC-движок остаётся доступен для выбора.

### Исправлено

- Кнопка «Перезапустить» после обновления на Linux теперь действительно
  перезапускает приложение — раньше из-за гонки с single-instance новый и
  старый экземпляры закрывались, и приложение не запускалось заново.

## [1.8.0] - 2026-06-14

### Добавлено

- Поддержка Windows: установщик `.exe` (NSIS) с движком GigaAM — голосовой
  ввод текста теперь работает и на Windows.
- Логирование в файл с ротацией по дням и запись паник в лог. На Linux —
  `~/.cache/arcanaglyph/logs/`, на Windows — `%LOCALAPPDATA%\...\logs\`.
  Нужно для диагностики, особенно на Windows (там нет консоли).

### Изменено

- Кнопка «Обновить» на Windows сама скачивает и запускает установщик `.exe`,
  после чего приложение закрывается для установки — раньше открывалась
  страница релиза в браузере и пакет приходилось ставить вручную.
- При обновлении на Linux окно терминала показывает пошаговую инструкцию
  (что происходит, когда вводить пароль `sudo`, что делать после установки) —
  раньше процесс выглядел как «зависание» на пустом экране.
- На Wayland разрешение на ввод текста (XDG RemoteDesktop) запрашивается при
  запуске приложения, а не при первом нажатии Ctrl+Ё — диалог «Дать доступ»
  больше не прерывает запись посреди работы. После первого согласия не
  повторяется.
- Регистрация горячих клавиш переведена на сравнение разобранных комбинаций
  вместо строк — надёжнее, и чинит хоткеи на Windows.
- Файловые логи хранят последние 7 дней (старые файлы удаляются автоматически) —
  раньше копились без ограничения, накапливая записи со старых версий.

### Исправлено

- Захват микрофона адаптируется к формату устройства: если оно не принимает
  «моно / 48 кГц», берётся его родной формат (стерео/f32 и т.п.) со сведением
  в моно — чинит запись на Windows (WASAPI), где ранее микрофон не открывался.
- Бейдж модели на главном экране больше не показывает весь путь на Windows
  (пути с обратными слешами теперь обрабатываются корректно).
- Подсказки про микрофон стали платформо-зависимыми (на Windows — настройки
  звука Windows вместо linux-only `pavucontrol`/`amixer`).
- Вставка текста на Windows идёт через буфер обмена + Ctrl+V и работает при
  любой раскладке клавиатуры — раньше текст искажался (в Блокноте вставлялась
  одна буква).
- При обновлении на Linux кнопка «Перезапустить» остаётся заблокированной, пока
  установка в терминале реально не завершится — раньше её можно было нажать
  слишком рано и перезапустить старую версию.
- Убрана лишняя полоса прокрутки на странице меню в Windows-сборке.

## [1.7.8] - 2026-06-13

### Исправлено

- Баннер «Доступно обновление» больше не появляется, пока на GitHub ещё
  собирается установочный пакет: проверка теперь убеждается, что `.deb`
  реально выложен в релиз, и только тогда предлагает обновиться. Раньше
  нажатие «Обновить» в этом окне приводило к ошибке установки.

## [1.7.7] - 2026-06-13

### Исправлено

- Свежее обновление теперь видно сразу после перезапуска приложения: проверка
  при старте больше не пропускается из-за суточного интервала (раньше баннер
  мог появиться с задержкой до 24 часов).

## [1.7.6] - 2026-06-13

### Исправлено

- Приложение больше не закрывается аварийно: устранены паники при триггере записи
  до загрузки модели (показывается сообщение вместо краша), при «отравлении»
  блокировок на горячих путях и при запуске второй копии (занятый UDP-порт 9002).
- На GNOME одно нажатие Ctrl+Ё больше не срабатывает дважды (start+stop за
  миллисекунды) — добавлен дебаунс повторного триггера в пределах 250 мс.
- Таймер и индикатор уровня во время записи больше не «дёргаются» после
  паузы/возобновления.
- UI больше не залипает в состоянии «идёт запись», если событие завершения
  обработки потерялось: добавлена сверка с реальным состоянием движка.
- История транскрибаций открывается быстрее при большом числе записей.
- В настройках подсветка несохранённых изменений теперь появляется на том
  переключателе предзагрузки, который реально изменили (раньше всегда
  подсвечивался Vosk).

### Изменено (внутреннее, без изменения поведения)

- Масштабный behaviour-preserving рефакторинг кодовой базы: god-функции разбиты
  на тестируемые хелперы, выделены общий DSP-модуль и единая фабрика транскрайберов,
  добавлен архитектурный gate. Значительно расширено тестовое покрытие
  (frontend 0% → 95.8%, суммарно 32% → 78.1%). Поведение приложения не изменилось.

## [1.6.8] - 2026-05-07

### Исправлено

- Сборка `.AppImage` в CI: `linuxdeploy-plugin-gtk` падал на запросах
  pkg-config к библиотекам, которые он бандлит (SVG-renderer, pango,

  - `libgtk-3-bin` — утилита `gtk-query-immodules-3.0`.
  - `libgdk-pixbuf2.0-bin` — утилита `gdk-pixbuf-query-loaders`.

## [1.6.0] - 2026-05-06

### Добавлено

- Self-contained `.deb`-пакет: один артефакт работает на любом x86_64 Linux.
- В git закоммичена `assets/libs/libonnxruntime-noavx.so`.
- STT-движки переведены на cargo features: `gigaam` (default), `vosk`, `whisper`, `qwen3asr`.
- При локальной сборке cargo-features автоматически подбираются под CPU.
- Runtime AVX-check в Tauri main: авто-переключение на не-ONNX fallback.
- Generic auto-download модели активного транскрайбера при первом запуске.
- Авто-распаковка `.zip`-архивов моделей при скачивании через UI.
- Событие `download://extracting` для индикации фазы распаковки.
- Новое событие `engine://model-loading` (Tauri).
- Eager-preload модели в `save_config`: при смене движка в Settings.
- В главном окне UI отображается прогресс скачивания модели.
- Новая Tauri-команда `get_compiled_engines`.
- Авто-fallback на дефолтный движок если выбранный не включён в сборку.
- Визуальный picker позиции виджета записи в Settings → Основное.
- GNOME Shell extension `arcanaglyph-widget@arfi.tech` для Wayland.

### Изменено

- В runtime-зависимости `.deb` пакета добавлен `libayatana-appindicator3-1`.
- `TranscriberType::default()` теперь `GigaAm`.
- Models tab: per-model резолюция пути для Whisper-вариантов в `get_models`.
- Dropdown «Движок транскрибации» теперь disabled-стилем для движков без моделей.
- В дроп-дауне «Движок транскрибации» Whisper расщеплён на «Whisper Tiny» и «Large».
- Toast-предупреждение при выборе тяжёлой модели на CPU без AVX2.

### Исправлено

- Совместимость с Rust 1.95 / Clippy 1.95.
- Зависимость `arcanaglyph-core` в `arcanaglyph-app` теперь с `default-features = false`.
- Вставка текста на Linux X11 переведена на clipboard (`arboard`) + `Shift+Insert`.
- В `main()` добавлен явный вызов glib `g_set_prgname("arcanaglyph")`.
- Команда установки `.deb` в README заменена на `sudo apt install ./<deb>`.
- Whisper транскрибация на Intel Atom Tremont: фикс краша `-6`.
- whisper-rs обновлён 0.13.2 → 0.16.0 (whisper.cpp 1.7.1 → 1.8.3).
- whisper.cpp/ggml внутренние логи теперь маршрутизируются через `tracing`.

### Удалено

- `gigaam-tract` backend и модель GigaAM v3 FP32 (~846 МБ).

## [1.0.0] - 2026-04-07

### Добавлено

- Четыре STT-движка: Vosk, Whisper, GigaAM v3, Qwen3-ASR.
- GigaAM v3 — лучшая модель для русского (WER 8.4%, 225 МБ).
- Qwen3-ASR — мультиязычный движок (52 языка).
- Автоскачивание GigaAM v3 при первом запуске.
- Горячие клавиши: Ctrl+Ё (запись), Ctrl+Shift+Ё (пауза).
- Авторегистрация хоткеев в GNOME через gsettings (Wayland).
- Композер горячих клавиш с кнопками модификаторов.
- VAD — авто-стоп записи при тишине после речи.
- Удаление слов-паразитов (э, э-э, ээ, эм, мм).
- Настройки с тремя вкладками: Основное / Модели / Клавиши.
- Карточки моделей с описанием, размером, статусом и кнопкой скачивания.
- Встроенное скачивание моделей с прогресс-баром.
- Автоочистка старых записей (настраиваемый период).
- Автозапуск при входе в систему.
- Запуск в трей (start minimized).
- Меню трея: Открыть / Запись / Настройки / Выход.
- Иконка трея: белая (idle), красная (запись), оранжевая (пауза).
- История транскрипций в SQLite с пагинацией.
- Ретранскрибация записей другой моделью.
- Предзагрузка моделей — мгновенное переключение.
- Пул моделей с ленивой загрузкой.
- XDG-стандартные пути: модели, конфиг, кэш.
- UDP-скрипты для Wayland (автоустановка).
- Проверка конфликтов хоткеев в GNOME.
- Раздел «О приложении» в меню.
- Сборка `.deb` через Tauri v2.
