## 🏗️ Архитектура и Рефакторинг

### 1. Выделение бизнес-логики из `arcanaglyph-app` ✅ Сделано (в рамках Фичи 1)

- **Проблема:** Файл `main.rs` в `arcanaglyph-app` содержит много прямой логики
  управления состоянием (например, AVX-fallback, управление скачиванием).
- **Решение (фактическое):** Совет «перенести оркестрацию в `arcanaglyph-core`» был
  некорректен — `core` это library без знания о Tauri. Вместо этого `main.rs`
  (2686 → 218 строк) разбит на модули **внутри `app/`**:
  - `commands/{engine,platform,config_history,updater_cmds,hotkeys,widget_ext}.rs`
  - `models/{download,registry}.rs`
  - `setup/{bootstrap,app_setup,events}.rs`

### 2. Унификация интерфейса моделей ❌ Уже было сделано

- **Проблема:** Модели `GigaAM`, `Qwen3-ASR`, `Whisper` и `Vosk` имеют разные подходы
  к инициализации (кто-то требует 4 сессии, кто-то одну).
- **Реальное состояние:** Фабрика моделей уже существует — `engine::create_transcriber`
  (`crates/arcanaglyph-core/src/engine.rs:132`) возвращает `Arc<dyn Transcriber>`.
  Trait `Transcriber` (`crates/arcanaglyph-core/src/transcriber.rs:8`) — единый
  интерфейс. Аргумент про «4 сессии vs 1» — это внутренняя деталь Qwen3,
  инкапсулирована в `Qwen3AsrTranscriber::new`.

### 3. Обработка ошибок во фронтенде ✅ Сделано (в рамках Фичи 3, узкий slice)

- **Проблема:** Сейчас многие ошибки просто выводятся в консоль или через базовые `toast`.
- **Решение (фактическое):** Добавлен сериализуемый тип `ApiError { kind, message, hint }`
  в `crates/arcanaglyph-core/src/error.rs` с маппингом из `ArcanaError`. Frontend
  получает структурированный payload через `engine://error` event и показывает
  hint-подсказку («Проверьте микрофон в pavucontrol», «Освободите ≥ 2 ГБ на диске»
  и т.п.). Helpers `errorMessage` / `errorHint` / `isCancelled` —
  в `frontend/src/shared/lib/tauri.ts`. Cancelled-ошибки теперь не показывают toast.

## 🚀 Производительность и Ресурсы

### 1. Оптимизация ONNX сессий ✅ Сделано (Фича 2)

- **Проблема:** Модели (особенно Qwen3) занимают много RAM.
- **Решение:** LRU-выгрузка моделей из пула. Новая настройка
  `model_unload_after_minutes` в `CoreConfig` (0 = выключено). Фоновый sweeper в
  `ArcanaEngine` раз в минуту выгружает неактивные модели; активная никогда не
  выгружается. Защищён от гонок с инференсом через `is_busy.try_lock` и
  `Arc::strong_count == 1`. Метод `loaded_models_idle_seconds()` отдаёт UI текущий
  простой каждой модели для отображения в Settings.

### 2. Pseudo-streaming для всех движков 📋 В Roadmap

- **Проблема:** Сейчас только Vosk поддерживает честный streaming. Для остальных
  пользователь ждет окончания записи.
- **Статус:** `Roadmap.md` Phase 8, приоритет 4. Pseudo-streaming для GigaAM CTC
  через накопительный inference — план есть, ждёт своей очереди.

### 3. Кэширование Mel-спектрограмм ⏸ Отложено

- **Проблема:** При повторной транскрибации той же записи (retranscribe)
  спектрограмма вычисляется заново.
- **Почему отложено:** Слабая ценность. Retranscribe — редкая операция (пользователь
  делает 1-2 раза за сессию максимум), а GigaAM mel ≈ 50 мс на 5 сек аудио. Выигрыш
  не оправдывает инвалидацию кэша при смене модели и дополнительные ~10 МБ на запись.

## 🎨 UX и Системная интеграция

### 1. Поддержка PipeWire 📋 В Roadmap

- **Проблема:** `cpal` хорошо работает с ALSA, но на современных дистрибутивах
  (Fedora, новейшие Ubuntu) интеграция с PipeWire через слой совместимости может
  быть менее стабильной.
- **Статус:** `Roadmap.md` Phase 10.4. На текущих Ubuntu 24.04 / Fedora 40 `cpal`
  через ALSA-compat работает стабильно — нативный PipeWire backend это
  nice-to-have, а не критичная задача.

### 2. Расширение глобальных хоткеев ⏸ Отложено

- **Проблема:** На Wayland/GNOME регистрация хоткеев через gsettings — это хак,
  требующий перезагрузки shell в некоторых случаях.
- **Почему отложено:** `tauri-plugin-global-shortcut` уже подключён
  (`main.rs:17`), но на Wayland нативно не работает. XDG Portal Global Shortcuts
  API всё ещё считается experimental в большинстве DE. Текущий путь через gsettings
  + кириллические дубли + setxkbmap-пинок (для X11) — стабильно работает на всех
  GNOME-сессиях. До тех пор пока Portal API не созреет — менять стек невыгодно.

### 3. Автоматическая пунктуация (Post-processing) 📋 В Roadmap

- **Проблема:** Whisper и Vosk выдают «сырой» поток слов.
- **Статус:** `Roadmap.md` Phase 9, **приоритет 1**. Лёгкая модель пост-процессинга
  (silero-te или Sage-CRF) для расстановки запятых и точек — в плане ближайших.
  GigaAM v3 проставляет пунктуацию сам, для остальных движков — TODO.

## 🛠️ Инструментарий и CI/CD

### 1. Автоматизация тестов для моделей ⏸ Отложено

- **Проблема:** Сейчас тесты в `core` покрывают только логику Mel и конфиги, но
  не сам инференс (так как нужны модели).
- **Почему отложено:** «Крошечная» модель-пустышка не тестирует реальный пайплайн.
  Whisper Tiny — 75 МБ, Vosk Small — 50 МБ; их скачивание в каждом CI-run стоит
  больше, чем тест ловит ошибок. Более ценно — integration test на установленном
  локально dev-окружении (что у нас и делает `make run`).

### 2. Улучшение `arcana-codex` индексации ❌ Вне скоупа

- **Проблема:** (Из REPORT.md) `affected_by` не видит `match`-блоки.
- **Почему вне скоупа:** ArcanaCodex — это **отдельный sister-проект**
  (`~/projects/ArFi_Tech/ArcanaCodex/`), который handoff'нут в свою сессию.
  Улучшения индексации делаются там, не здесь. Этот репо (`arcanaglyph`) только
  использует `arcana-codex` как MCP-клиент для навигации по коду.
